# Chat Init

## 目的

このファイルは、`pj-general` で毎チャットの最初に読む共通初期化ファイルである。

ここには、現状の handoff やセッション依存情報ではなく、毎回共通で効く起動前提だけを書く。

## 起動時の基本動作

1. `AGENTS.md`
2. `PROJECT.md`
3. この `chat-init.md`

の順で読み、その後に必要な `docs/*` を読む。

## 毎チャットの共通前提

- 日本語で対応する
- すべて UTF-8 として扱う
- この PJ 固有の更新は `G:\devwork\pj-general` を正本にする
- 横断価値のある知見だけ `G:\knowledge-vault` の既存カテゴリへ反映する
- 新しい知見倉庫は増やさない
- まだ要件定義前フェーズなので、仕様断定より情報設計、運用設計、比較材料整備を優先する

## 更新時の共通ルール

- PJ 固有の設計、比較、進行状態は `pj-general` 側 docs を正本にする
- セッション記録は `docs/diary/` に置く
- 実装者向けの進行管理は `docs/imp/imp-*` に置く
- ユーザーが見るべき確認事項は `docs/imp/user-*` に置く
- product / arch / data / org など、責務に応じた docs 配置を崩さない

## 技術検討時の共通ルール

- 最新情報や外部 OSS の仕様は、必要なら一次情報で確認してから扱う
- OSS は「丸ごと採用」ではなく「どのレイヤーを借りるか」で評価する
- 大規模化を前提に、境界、依存、将来の分割可能性を優先する
- 独自価値がある入口体験は自前で握る前提を崩さない

## 出力時の共通ルール

- 結論を先に出す
- 次の行動が分かる形で短くまとめる
- 長い検討では、判断、未解決論点、次アクションを分けて書く

## 現状把握の読み先

現状の進行や handoff が必要なときは、次を優先して読む。

- `docs/diary/*`
- `docs/imp/user-tasks.md`
- `docs/imp/user-judge.md`
- `docs/imp/imp-tasks.md`
- `docs/imp/ai-review-brief-*.md`

## このファイルに書かないもの

- セッション固有の要約
- 一時的な判断待ち
- 特定日の handoff
- 現時点だけ有効な検討メモ

これらは `docs/diary/` や `docs/imp/` に置く。
