# 要件定義質問集 2026-06-29

## 目的

この質問集は、`pj-general` の要件定義を前へ進めるためのユーザー向けヒアリングシートである。

前回の 50 問版から、すでに方向性が固まった論点を落とし、今も設計を止めやすい論点だけ 20 問に絞った。
移動中でも答え続けやすいように、短文回答で進められる形にしている。

## 今回の前提

- `入口は自前実装`
- `4 窓口構成`
- `Plane / OpenProject / Leantime を役割分担して参照`
- 技術スタック第一候補は `tech-stack.md` を正本とする

今回の質問は、主に次を詰めるためのものに限定する。

- 正本 object
- 同期方向と冪等性
- AI 分別の確定方式
- 権限の最小モデル
- MVP と後段要件の切り分け

## 回答ルール

- 1 問あたり 1 行から数行でよい
- 未定なら `未定`
- 迷う場合は `第一候補 / 第二候補`
- ID を残して答える

例:

```text
AA-00: 最初は自分中心。ただし外部協力者を早めに入れたい。
AA-01: Slack と手入力から入った雑な情報を、放置せずタスク化まで進めたい。
```

## 質問 20 選

### A. 初期版の中心価値

- `AA-00` 初期版の主利用者は誰か。`自分のみ`、`自分 + 外部協力者`、`最初から小チーム` のどれで始めるか。
- `AA-01` 初期版で最も解決したい 1 つの困りごとは何か。
- `AA-02` 初期版で「使い続ける価値がある」と判断できる最低ラインは何か。
- `AA-03` 初期版で絶対に外したくない流れは何か。`入力 -> 分類 -> タスク化 -> 予定化` のどこまでを必須にするか。

### B. 正本 object と昇格ルール

- `AA-04` `アイデアカード` は `タスク` と別 object として残したいか。それとも初期版では同系列でよいか。
- `AA-05` `検討事項` と `気になっている事` は分けたいか、ひとまず同じ箱で扱いたいか。
- `AA-06` 1 つの入力から `タスク` と `予定` の両方が生まれることを初期版から許容したいか。
- `AA-07` `アイデア -> タスク` に昇格できる最低条件は何か。
- `AA-08` `タスク -> 予定` に昇格できる最低条件は何か。

### C. 外部入口と同期

- `AA-09` 初期版の入口優先順位を並べるとどうなるか。`手入力`、`Slack`、`knowledge-vault` の順番で答えてほしい。
- `AA-10` Slack では何を回収対象にしたいか。`自分の全投稿`、`特定チャンネルのみ`、`印付き投稿のみ` など。
- `AA-11` knowledge-vault では何を回収対象にしたいか。カテゴリ、ファイル種類、更新条件など。
- `AA-12` 同じ内容が複数入口から入ったとき、何を正本にしたいか。
- `AA-13` 取り込み後に元リンクや出典を必ず保持したいか。

### D. AI 分別と自動化

- `AA-14` AI 分別の初期方針はどれにしたいか。`即確定`、`下書き保存`、`人レビュー必須`。
- `AA-15` AI に最初から任せたい範囲はどこまでか。`分類のみ`、`要約まで`、`タスク案生成まで`、`予定案生成まで`。
- `AA-16` 定期回収の頻度は本当に `1 時間ごと` でよいか。別案があればそれも答えてほしい。

### E. 権限と画面

- `AA-17` 外部協力者には、初期版で `何を見せるか` と `何を編集させるか` を分けて答えてほしい。
- `AA-18` 4 窓口のうち、初期版で最優先なのはどれか。`横断ダッシュボード`、`管理画面`、`書き入れ口`、`実績 / 履歴参照`。

### F. MVP と後段

- `AA-19` 次の 5 つを `初期版に入れる / 後段に回す` で仕分けてほしい。
  - Google Calendar 直接登録
  - ガントチャート
  - キャパ管理
  - Codex プロジェクト開始支援
  - 外部協力者向け権限

## この 20 問の使い方

- まず `AA-00` から順に答える
- 迷う設問は `未定` で飛ばしてよい
- 回答後は、その結果をもとに `docs/spec`、`docs/data`、`docs/org` の初版へ落としていく
