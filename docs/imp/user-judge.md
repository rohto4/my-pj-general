# ユーザー判断待ち

## 今回のヒアリングで次に確認したいこと

- Google Calendar 連携をどこまで自動化したいか
- 外部協力者に最初から見せる情報の範囲
- MVP にガントチャートを入れるか
- MVP にキャパ管理を入れるか
- Codex プロジェクト起動支援を独立機能にするか
- 4 つの窓口ごとに何を表示し、何を操作可能にするか
- Slack と knowledge-vault の回収後、AI 自動分別をどこまで自動確定させるか

## 現時点の仮説

- MVP は「アイデアカード -> タスク化 -> 予定化」の一気通貫が中心
- ガントとキャパ管理は第2段階に寄せた方が、初期価値を早く検証しやすい
- ロール管理は MVP 必須

## 方針更新

- ユーザーは現時点で実装順より要件整理の深さを優先している
- 以後の質問は、MVP に絞り込むためだけでなく、将来要件も含めて広めに拾う
