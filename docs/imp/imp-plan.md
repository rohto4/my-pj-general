# 実装方針メモ

## フェーズ 0: 地盤整備

- ルート構造の初期化
- テンプレート、skills、commands の流用
- docs 運用の初期化
- 外部 repo を参照しつつ、Codex で skill を内製追加できる体制を整える

## フェーズ 1: 要件前整理

- ユースケース整理
- 利用者ロール整理
- データの大分類整理
- 候補 OSS の評価軸確定

## フェーズ 2: OSS 選定と統合方針

- 候補 OSS の比較
- コア採用対象の仮決定
- UI 統合 / バックエンド分離 / データ同期の方針決定
- 外部土台だけで足りない作業を、どの skill として内製するか決める

## フェーズ 3: MVP 設計

- MVP の最小ユースケースを固定
- 境界ごとの責務を定義
- 実装順を決める
