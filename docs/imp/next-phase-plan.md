# 次走の進め方

## 目的

候補 OSS を評価し、この PJ に最適な引用・拡張方針を決める。

前提として、この PJ は当面 Codex 専用で進める。

## 進め方

1. 参考記事に載る候補 OSS と、その周辺の有力 OSS を一覧化する。
2. 技術スタック、ライセンス、セルフホスト性、API、UI 統合しやすさを一次情報で確認する。
3. 次の 3 系統に分けて比較する。
   - プロジェクト / チケット管理の中核候補
   - ナレッジ / ドキュメント統合候補
   - DB / 業務アプリ拡張候補
4. 統合パターンを複数描く。
   - 単一 OSS を母体に拡張
   - 独自 UI + 複数 OSS バックエンド
   - 独自コア + OSS を補助採用
5. MVP に必要な境界だけ先に切り出して、採用候補を絞る。

## 今回やらないこと

- MiMo / GLM を含むマルチエージェント互換設計
- agent profile の共通化
- マルチエージェント用 MCP 基盤の汎用設計

## 成果物

- 候補 OSS 比較表
- 採用候補ランキング
- 統合シナリオ案 2〜4 本
- MVP の実装順ラフ案
