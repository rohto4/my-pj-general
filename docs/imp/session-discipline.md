# セッション運用メモ

## 方針

- 長いセッションでも、会話だけで進めず docs に都度反映する
- ヒアリング結果は `docs/product` に寄せる
- OSS 比較や参照候補は `docs/candi-ref` に寄せる
- 次に確認したいことは `docs/imp` に寄せる
- 運用ルールや進め方は `AGENTS.md` / `PROJECT.md` 側で管理する
- `knowledge-vault` は、この種のセッション運用ルールの置き場にはしない

## 現在の未完了事項

- ガント / スケジュール表示に使う OSS やライブラリの借用境界を詰める
- 横断ダッシュボードの思想参照元をさらに狭める
- 実績データとして保持すべき履歴項目を洗い出す
- Codex プロジェクト開始支援の入力項目と出力物を整理する

## 運用ルール

- 大きな判断が出たら、その場で 1 ファイル以上に反映する
- 次アクションが見えたら `docs/imp` に残す
- セッションが長引いたら要約より先に md を更新する
- ルール変更は AGENTS / PROJECT に寄せ、検討メモと混ぜない
