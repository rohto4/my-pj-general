最小読込セット

共通初期化の後、作業別に設計書を選び、足りないときだけ実装へ進む。

  1. 共通: AGENTS.md / PROJECT.md / tech-stack.md / README.md / docs/imp/*tasks.md
  2. 対象行の「正本」を読む。
  3. 受入・回帰の根拠を読む。
  4. 設計書にない事実だけを、表示した実装境界から確認する。
  5. 不足を本表・imp-tasksへ記録し、正本へ補完する。

読込を広げる前の判断

コードを先に検索してよいのは、設計書が対象の状態・操作・境界を断定できないときだけ。画面の細かなselector、上流forkの制約、API payloadの実測、再配信script変更は例外として対象ファイルだけを読む。

役割別の具体的なファイル順はimplementation-context-reading-guide.mdを正本とする。プロダクト判断を docs/imp の進捗メモで代用しない。

評価の読み方と更新ルール
  • 各列は「この機能を安全に変更・受入するために、実装を広く読まず正本から開始できる度合い」。対象外の列はその旨をstatusに明記して5とする。
  • 5: 正本・同期先・実装境界・回帰が一致。3-4: 正本はあるがP0の最新画面、実機証跡、運用境界が弱い。1-2: `docs/imp` の設計メモまたは実装だけに事実がある。
  • 仕様を確定したら product/spec/data/arch/ops の該当正本を更新し、本表はリンクと評価だけを同期する。進行・判断待ち・ユーザー操作は docs/imp に置く。
  • この表はP0/P1の完成度星取表とは別の、設計書の読込品質を測る正本である。
