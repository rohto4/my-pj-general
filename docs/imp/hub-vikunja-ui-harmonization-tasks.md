# Hub / Vikunja UI調和タスク

作成日: 2026-07-11

## 目的

Hubを入口・候補・判断のためのブルー + シルバー + ソリッドな業務画面として2段階洗練し、Tasks側のVikunja画面へ移動したときにも視覚的な不連続を感じにくくする。

データ結合、導線結合、P0の動作確認を優先し、画面の見た目だけを先行して仕様や責務境界を揺らさない。

## デザイン方針

- ブルーを操作・リンク・選択状態の主色にする。
- シルバー、白、薄いグレーを面・境界・非操作領域に使い、彩度を増やしすぎない。
- 操作可能、選択中、操作不能、表示専用を明暗と線で明確に分ける。
- 不要な丸角、装飾的な影、過剰なグラデーションを減らす。
- タイトル、本文、状態、操作の階層を保ち、文字を小さくして密度を稼がない。
- HubのカードやボタンはVikunjaの実画面を参照し、画面遷移後も主要な余白・境界・状態色が極端に変わらないようにする。

## 2段階の実装タスク

### 第1段階: Hubの視覚基盤

- [x] 色トークンをブルー、シルバー、選択、無効、警告、完了へ整理する
- [x] パネル、表、詳細、フォームの角丸と境界線を再設計する
- [x] ヘッダー、ナビ、Tasks側導線、確認待ち操作の優先順位を揃える
- [x] WQHDと中間幅で、文字の可読性と3列比率を確認する
- [x] 画面スクリーンショットを比較し、不要な余白・重複情報・視線移動を調整する

### 第2段階: Vikunjaとの調和

- [x] Vikunjaの実画面配信CSSから、サイドナビ、プロジェクト見出し、タスク一覧、状態表示の視覚要素を確認する
- [x] HubのTasks概要、Tasks側リンク、参考ガントの表現をVikunja側の情報階層へ寄せる
- [x] HubでのGO済み表示とVikunja側の未完了・完了表示の状態色を対応させる
- [x] HubからTasks側へ移動する外部リンクと、Hub内の戻り先を確認する
- [ ] Hub / Vikunjaの画面を並べてユーザー確認を行い、微調整項目を `docs/imp/user-tasks.md` に積む

## 完了条件

- Hubがブルー + シルバー + ソリッド基調で統一されている。
- 操作可能、選択中、操作不能、表示専用の区別が視覚的に明確である。
- HubのTasks概要とVikunjaのタスク画面で、タイトル・状態・期限・進捗の情報階層が調和している。
- WQHDと中間幅で、主要文字や操作ボタンが折り返し・重なりなく読める。
- 見た目の変更で、SQLite、GO、Webhook、再照合、Tasks側遷移を壊していない。
