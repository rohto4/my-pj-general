# OSS Selection Backlog

## 次走の対象

候補 OSS のうち、次の観点で比較する。

- アジャイル / スプリント管理
- カンバン / 個人タスク管理
- ドキュメント / ナレッジ連携
- マルチテナント、権限、ロール
- API / Webhook / 拡張性
- セルフホスト容易性
- UI 統合しやすさ
- ライセンスと改変しやすさ
- 既存コード規模、技術スタック、保守性

## 現時点の見立て

- 単一 OSS をそのまま採用するより、基幹 OSS + 独自 UI + 補助 OSS の組み合わせになる可能性が高い。
- UI 統合とデータ統合は分けて判断する必要がある。
- 要件次第で、プロジェクト管理 OSS とナレッジ / DB 系 OSS を併用する可能性が高い。
