# ユーザーの声ベースの比較メモ

## 方針

- 学術材料は使わない
- 一次情報とは別に、実ユーザーの声を補助材料として見る
- `ビュー数` は安定取得しづらいため、公開値として取得しやすい `レビュー件数` と `レビューの新しさ` を代用シグナルとして扱う

## 参照元

- G2 のレビュー一覧

## 1. OpenProject

- G2 評価: `3.8/5`
- レビュー件数: `25`
- 直近に確認できたレビュー日: `2025-10-08`, `2025-04-08`

ユーザーの声:

- 良い点:
  - タイムライン、ボード、タスク追跡を 1 か所で見られる
  - 役割やカテゴリに合わせて work package を分けやすい
  - 複数管理環境で使いやすい
- 気になる点:
  - 初期セットアップが重い
  - 検索は改善余地あり

今回への示唆:

- `全体管理`, `ガント`, `カテゴリ分け`, `役割別整理` の相性は良い
- ただし、あなたが自前で握りたい入口部分とは別物
- 借りるなら `横断ダッシュボード` と `ガント周辺の情報設計`

## 2. Cal.com

- G2 評価: `4.6/5`
- レビュー件数: `153`
- 直近に確認できたレビュー日: `2026-03-04`, `2025-12-05`, `2025-12-04`

ユーザーの声:

- 良い点:
  - back-and-forth をなくせる
  - double booking 防止
  - カスタマイズ性が高い
  - Google Calendar や Zoom 連携が自然
  - availability, buffer, limits の扱いが便利
- 気になる点:
  - 初期セットアップはやや複雑

今回への示唆:

- `スケジュール管理` ではかなり強い
- 予定調整、空き時間、予約導線の思想を借りる価値が高い
- プロジェクト管理の土台ではなく、`予定化` と `共有スケジュール` の参考元

## 3. Leantime

- G2 評価: `4.7/5`
- レビュー件数: `3`
- 直近に確認できたレビュー日: `2023-07-05`

ユーザーの声:

- 良い点:
  - `ideas -> goals -> todos` を 1 つの流れで扱える
  - Kanban と Gantt の切替が自然
  - アイデアを details と task に落としていきやすい
- 気になる点:
  - 若い OSS らしい成長痛への言及がある
  - 直近レビュー量は少ない

今回への示唆:

- `書き入れ口 / 作成口` の発想にはかなり近い
- ただし、市場シグナルはまだ弱い
- 借りるなら `アイデアから行動へ育てる導線`

## 4. Plane

- G2 の `Plane` は今回対象の project management OSS ではなく、別カテゴリ製品がヒットしたため除外
- Plane についてユーザーの声を見るなら、別のレビュー導線や動画を個別に探す必要がある

今回への示唆:

- 公式の AI / Intake / Pages の魅力は強いが、現時点ではユーザー声の補助材料が足りない
- そのため、Plane は `一次情報で強いが、二次情報確認が未完了` の状態

## 5. Redmine

- G2 評価: `4.0/5`
- レビュー件数: `257`
- 直近に確認できたレビュー日: `2026-04-10`, `2026-03-30`, `2026-03-10`

ユーザーの声:

- 良い点:
  - シンプルで self-host しやすい
  - Gantt と time tracking が使いやすい
  - チケット、文書、レポートを一通りまとめやすい
  - 無料で基本を押さえられる
- 気になる点:
  - UI が古い
  - プラグインの鮮度問題
  - 高度な sprint 管理や reporting は弱い
  - 権限や画面密度の面で情報過多になりやすい

今回への示唆:

- `古典的だが実利用の厚い基準点` として価値が高い
- `ガント`, `チケット`, `時間管理`, `self-host` を重視する比較対象として残すべき
- 借りるなら `課題管理の骨格` と `基本的な Gantt / Calendar の載せ方`

## 6. Tuleap

- G2 評価: `4.3/5`
- レビュー件数: `20`
- 直近に確認できたレビュー日: `2026-01-14`, `2025-12-24`, `2025-12-01`, `2025-10-29`

ユーザーの声:

- 良い点:
  - 柔軟性と設定自由度が高い
  - 複数ツールが統合されていて traceability が強い
  - 権限管理が細かい
  - 大きめの組織や規制寄り開発にも向く
- 気になる点:
  - UI / UX は初見に重い
  - 新規ユーザーにはガイドが必要
  - OnPrem 運用には DevOps / Linux 管理スキルが要る
  - ログや一部管理機能は改善余地あり

今回への示唆:

- `PJ管理 + 権限 + トレーサビリティ` の厚い比較対象として有効
- ただし、あなたが欲しい軽い入口とは距離がある
- 借りるなら `権限の考え方`, `複数成果物をつなぐ設計`, `管理画面の思想`

## 7. ProjectLibre

- G2 評価: `4.2/5`
- レビュー件数: `36`
- 直近に確認できたレビュー日: `2024-02-28` を確認

ユーザーの声:

- 良い点:
  - Gantt が主目的でわかりやすい
  - cost-effective
  - タスク依存、マイルストーン、リソース割当の基本が揃う
  - MS Project 代替として理解しやすい
- 気になる点:
  - コラボレーションは弱い
  - レポートの硬さ
  - フリーズやクラッシュ言及
  - 新規ユーザーには少し難しい

今回への示唆:

- `ガント専用寄りの見せ方` の参考として有効
- プロダクト全体の土台ではなく、`ガント画面の参考元` に向く
- 借りるなら `ガントの列構成`, `依存関係の見せ方`, `リソース割当の表示`

## 現時点の補助結論

### ユーザーの声が比較的厚い

- `Cal.com`
- `OpenProject`
- `Redmine`

### 発想は良いが、声の量は薄い

- `Leantime`
- `Tuleap`
- `ProjectLibre`

### 一次情報は強いが、二次情報の補強が必要

- `Plane`

## 今回の使い方

- `OpenProject`: ガント / 横断ダッシュボードの主参照
- `Cal.com`: スケジュール管理の主参照
- `Leantime`: 入口導線の発想参照
- `Redmine`: 古典的な実利用基準線
- `Tuleap`: 権限とトレーサビリティの厚い比較対象
- `ProjectLibre`: ガント専用寄りの比較対象
- `Plane`: 継続調査
