# デモ比較チェックリスト

## 対象

- OpenProject
- Plane
- Cal.com

## 目的

- 画面を実際に見ながら、何を借りるかを判断しやすくする
- 「丸ごと採用するか」ではなく「どのレイヤーを参考にするか」を見極める

## 使い方

- 各サービスを触りながら、項目ごとに `強い / 普通 / 弱い / 対象外` で見る
- 良かった点と、自前実装に取り込みたい点を一言ずつ残す
- 特に UI の第一印象ではなく、情報構造と操作の流れを重視する

## 1. 入口 / 作成導線

見る点:

- 新規入力の入口は見つけやすいか
- アイデアや雑メモを受け止められるか
- タスク化の流れは自然か
- 1つの入力から情報を育てていけるか
- AI 補助がある場合、入力体験を壊していないか

今回の主眼:

- `Plane` を主参照候補
- `OpenProject` は比較対象
- `Cal.com` は基本対象外

## 2. プロジェクト管理の骨格

見る点:

- プロジェクト、タスク、担当、期限、状態の関係がわかりやすいか
- 複数人・複数タスクをまとめて見やすいか
- 上位のまとまりと下位タスクの関係が追いやすいか
- 権限やロールの存在が自然に感じられるか

今回の主眼:

- `OpenProject` を主参照候補
- `Plane` を比較対象
- `Cal.com` は対象外寄り

## 3. ガントチャート / タイムライン

見る点:

- 一目で全体進行が把握できるか
- タスクの依存や重なりが読み取りやすいか
- 表示密度は高いか、うるさすぎないか
- 横スクロール、ズーム、期間切替が扱いやすいか
- ドラッグ操作が必要以上に複雑でないか

今回の主眼:

- `OpenProject` を主参照候補
- 補助参照として `GanttProject` と `ProjectLibre`

## 4. スケジュール管理 / カレンダー

見る点:

- 予定の見やすさ
- 人ごとの予定や空き状況の扱いやすさ
- 予定登録までの流れ
- shared availability の思想があるか
- タスク由来のスケジュールと、会議由来のスケジュールがどう分かれているか

今回の主眼:

- `Cal.com` を主参照候補
- `OpenProject Team Planner` を比較対象

## 5. 人員キャパ / 負荷の見え方

見る点:

- 誰が詰まっているか一目でわかるか
- 負荷の見せ方が時間基準か、件数基準か
- 色やレイアウトで直感的に読めるか
- 管理者視点と担当者視点の差があるか

今回の主眼:

- `OpenProject Team Planner`
- `Cal.com` の shared availability は補助参考

## 6. ダッシュボード性

見る点:

- 最初の画面で何を把握できるか
- 「次に何をすべきか」が伝わるか
- 横断ビューとして使えるか
- 閲覧中心画面と管理中心画面が分かれているか

今回の主眼:

- `OpenProject`
- `Plane`

## 7. 管理画面の切り分け

見る点:

- 一般利用者向け画面と管理者向け画面が混ざっていないか
- 権限設定や組織設定がどこにあるか
- 強い操作を日常画面に出しすぎていないか

今回の主眼:

- `OpenProject`
- `Plane`

## 8. 実績 / 履歴データの前提

見る点:

- ステータス履歴が残りそうか
- 完了や変更の履歴を扱える思想があるか
- 後から見せ方を増やせるだけのデータ前提がありそうか

今回の主眼:

- `OpenProject`
- `Plane`

## 9. 自前で握るべきか、借りるべきか

各対象ごとに最後にこれを判定する。

- そのまま借りたい
- 情報設計だけ借りたい
- UI の見せ方だけ借りたい
- 発想だけ借りたい
- 今回は借りなくてよい
