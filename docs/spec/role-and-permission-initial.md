# P0 ロール・権限初期仕様

作成日: 2026-07-10

## 目的

P0 の画面と操作に対するロール・権限の初期方針を定義する。

P0 は個人運用を先に成立させるため、実認証や多人数権限制御は作らない。ただし、画面設計とデータ構造は後で外部協力者を追加できる前提にする。

## 初期ロール

| ロール | 目的 | P0 表示 |
| --- | --- | --- |
| Owner | 自分。全入口、全候補、全判断を扱う | 全画面、GO / 編集 / 不要 / アーカイブ可 |
| External collaborator | 外部協力者。予定中心で関わる | スケジュール中心表示。詳細タスクや内部判断は制限候補 |

## P0 操作権限

| 操作 | Owner | External collaborator |
| --- | --- | --- |
| 書き入れ | 可 | 後続 |
| 確認待ち閲覧 | 可 | 後続。schedule 中心に制限 |
| GO | 可 | 不可または要承認 |
| 編集 | 可 | 後続 |
| 不要 / アーカイブ | 可 | 不可 |
| ガント閲覧 | 可 | schedule 中心で可 |
| 管理画面 | 可 | 不可 |

## P0 で作らないこと

- 認証。
- ユーザー招待。
- 組織管理。
- 行レベル権限。
- 外部協力者専用 route。

## 管理画面で見る項目

- ロール名。
- 閲覧範囲。
- GO 可能か。
- schedule 中心表示か。

## 実装対応

- UI: 管理画面 `ロール表示`
- P0 DB: 現時点では `settings` または将来 `roles` table へ分離
